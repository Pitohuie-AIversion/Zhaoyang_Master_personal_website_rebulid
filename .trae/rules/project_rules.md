# 牟昭阳个人学术网站项目规则

## 语言与沟通
- 默认使用中文交流；需要英文输出时请显式说明。
- 回答中保持简洁专业，避免与项目无关的闲聊。
- 涉及学术术语时，优先使用国际通用英文术语，确保专业性。

## 多语言/翻译规范
- 所有界面文案必须通过 `t('<命名空间>.<键>')` 获取，不得在组件中硬编码中英文字符串。
- 为每个新增的页面或组件同时在 `src/locales/zh.json` 和 `src/locales/en.json` 中补齐键值，并确保键名一致。
- 新增翻译键时优先按页面或模块划分命名空间：
  - `home.*` - 首页相关
  - `research.*` - 学术研究页
  - `projects.*` - 项目作品页
  - `skills.*` - 技能专长页
  - `contact.*` - 联系我页
  - `common.*` - 通用组件和文案
- 学术专有名词（如论文标题、期刊名称、技术术语）保持英文原文，不做翻译。
- 修改或删除翻译键前，需全局搜索确认不会影响其它页面。

## 学术内容规范
- 论文信息必须包含：标题、作者、期刊/会议、年份、DOI、摘要等完整信息。
- 专利信息必须包含：名称、专利号、申请日期、公开日期、发明人、摘要等。
- 项目展示需包含：项目名称、时间、技术栈、项目描述、关键成果等。
- 所有学术内容数据必须真实准确，不得虚构或夸大。

## 代码与结构
- 组件内的文案、枚举或选项值使用稳定的代码（如 `status`、`category` 等），通过映射表获取当前语言显示文本。
- 共用组件（如搜索、筛选、模态框）不得保留硬编码文案，全部改用翻译键并在 locale 中维护。
- 新增学术数据项（论文、专利、项目等）时，先定义数据结构中的键，再同步补充翻译内容。
- 保持学术内容的数据结构一致性，便于后续维护和扩展。

## 性能与体验
- 学术内容（论文、专利等）图片需优化压缩，确保加载速度。
- 移动端适配必须完整，确保学术内容在各种设备上可读性良好。
- 搜索和筛选功能需支持中英文内容检索。

## SEO与可访问性
- 每个页面必须有完整的 meta 标签，包含中英文描述。
- 图片必须包含 alt 属性，描述图片的学术内容。
- 确保网站符合 WCAG 2.1 标准，支持屏幕阅读器。

## 提交流程
- 在提交前自查：
  - 语言切换是否正确显示所有学术内容
  - 是否存在未补齐的翻译键
  - 是否有重复或被覆盖的命名空间
  - 学术内容数据是否完整准确
  - 新增内容是否遵循了学术规范
- 推荐在 PR 说明中列出修改过的主要翻译键及影响的页面，方便 Review 与测试。
- 涉及学术内容修改时，需提供相关证明材料或来源链接。
